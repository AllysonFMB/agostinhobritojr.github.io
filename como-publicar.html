1. A publicação e seus objetivos

Embora os conteúdos oferecidos nas disciplinas dos cursos de graduação seja essenciais para a formação do engenheiro, certamente não conseguem abordar toda a gama de conhecimentos possíveis que se encontram disponíveis por aí. Normalmente, seus interesses em assuntos específicos guiam sua curiosidade para aprender mais que o normal sobre um determinado assunto. É neste sentido que compartilhar seu conhecimento com a comunidade acadêmica pode ser importante.

Qual a recompensa pela sua contribuição?

A princípio, Nenhuma!!! Nada de dinheiro, pontos extras ou a glória eterna...

Entretanto, descrever suas experiências com um determinado tipo de tecnologia com a qual você possui familiaridade é importante sob diversos aspectos que podemos elencar.

Um deles é motivar seus colegas a conhecer algo novo, fomentando a busca pelo conhecimento. Além disso, descrever sua experiência para outros demonstra seu interesse pela ciência e pode abrir portas para pessoas interessadas em profissionais com sua formação.

Outro aspecto é o da exposição à qual seu documento ficará sujeita. A princípio, postar um artigo ou tutorial para o público pode parecer intimidante, dadas possíveis críticas que podem ser recebidas. Entretanto, é uma experiência valorosa, pois você se verá forçado escrever com cautela, ponderando as sentenças do seu texto de modo a fornecer uma leitura agradável. Além disso, as críticas enviadas servirão para que seu documento possa ser aperfeiçoado e, inclusive, permitir que você aprenda formas mais adequadas de lidar com assuntos que julgava conhecer bem.

Por fim, a interação social promovida pelo texto que você publica permite que você aperfeiçoe a prática de expor ideias e escrever para terceiros, além de poder fomentar o desenvolvimento de projetos pessoais que poderão lhe render bons frutos, algo bastante comum no ambiente de trabalho de um engenheiro.

2. Regras a obedecer

Ao submeter um texto para publicação, você deverá concordar em ser total responsável pelo conteúdo presente nesse texto.

Quando se prepara documento para publicação em uma comunidade de abrangência global, uma série de cuidados devem ser tomados para que seu texto produzido não seja ofensivo aos possíveis leitores do conteúdo.

Você é responsável pela sua publicação! Caso contenha violação de direitos autorais, você poderá ser processado na forma da Lei pelas violações cometidas. Sendo assim, é importante obedecer algumas regras simples que podem evitar problemas.

A primeira delas (e a mais importante) é: NUNCA COPIE NADA!

Obedeça esta regra cegamente. Tome-a como primeiro mandamento de seu código de conduta. NUNCA copie textos, figuras, imagens, vídeos, equações, ou quaisquer outros materiais criados por terceiros. Preste sempre atenção às limitações impostas pelos direitos autorais. Via de regra, se você prepara seu texto sem olhar ou se basear no de outros autores, as chances de você infringir direitos autorais são mínimas. Em se tratando de figuras, imagens, vídeos, etc., crie os seus próprios materiais, pois dificilmente você conseguirá cópia de algo que já existe publicado.

Claro, existem diversos websites que disponibilizam recursos para uso livre. Nada o impedirá de utilizá-los. Verifique apenas a licença de uso dos recursos e tipos de informações que seu documento deverá conter para que possa incluí-los de forma que é permitida. Geralmente, a inclusão de conteúdos é permitida mediante a citação expressa da fonte e/ou de alguma licença de uso.

3. Ferramentas disponíveis

Em se tratando de uma publicação via web, a forma mais comum de preparar o texto para publicação é em HTML. HTML (Hypertext Markup Language) é uma linguagem de especificação de documento criada especificamente para realizar a diagramação de páginas web. Não é difícil aprender html. Entretanto, ao contrário de um editor de texto com diagramação visual (tal como Office ou LibreOffice) onde é possível descobrir funcionalidades de uso na própria interface do programa, é bastante razoável ter à mão um tutorial para iniciar e, posteriormente, um guia de referência os marcadores que a linguagem disponibiliza.

Enquanto não conseguimos disponibilizar um tutorial de html, você poderá encontrar um bom material para aprender a linguagem em http://en.wikibooks.org/wiki/HTML.

Existe também opções em português disponíveis, como esta aqui. Uma busca rápida no Google poderá ajudá-lo a encontrar outras opções...

A diagramação é feita em HTML, mas as ilustrações devem ser feitas utilizando ferramentas apropriadas. Geralmente, as páginas aqui apresentadas são desenvolvidas usando software livre. Entre elas, destacamos as seguintes:

Docbook (www.docbook.org)
usada para confecção de tutoriais. Facilita a diagramação e geração automática de um documento em múltiplas páginas, além de permitir uma boa integração com folhas de estilo que enriquecem a qualidade visual das páginas geradas.
Gimp (www.gimp.org)
usada para manipulação de imagens digitais. Permite abrir diversos tipos de formatos de imagens diferentes, tratá-las com filtros digitais e experimentar diversos tipos de efeitos visuais diferentes.
Inkscape(www.inkscape.org)
usada para criação de ilustrações vetoriais. Navegadores como Firefox e Google Chrome renderizam SVG corretamente, permitindo a exibição de ilustrações com alta qualidade visual.
Emacs(www.gnu.org/s/emacs)
usado como editor de texto. Possui modos de edição especiais que facilitam a tarefa de programar não só em HTML, mas em diversas outras linguagens de programação.

O desenvolvimento dos materias apresentados neste site é feito em sistemas Linux.

4. Como submeter sua publicação

Oferecemos uma página como MODELO de referência para que você possa publicar seus artigos seguindo um padrão comum a todos. Você pode recuperar o diretório completo em um único arquivo .zip (modelo.zip).

A princípio, estamos enquadrando as publicações em três linhas principais:

Tutoriais
Documentos extensos que tratam com maior profundidade acerca de um determinado assunto, detalhando aspectos de uso de uma dada tecnologia.
Artigos
Textos descrevendo como uma determinada tecnologia foi utilizada para resolver um problema, ou para desenvolver um produto ou experimento.
HowTos
Guias de configuração/instalação de uma determinada ferramenta, detalhando uma experiência de uso. (Ex: como preparar o ambiente de edição e testes de documentos HTML em um dado sistema operacional.)

Pode ser comum que uma mesma proposta de documento seja oferecida para publicação mais de uma vez. Logo, recomendamos que se dê uma vasculhada nos textos que já foram publicados para evitar superposições.

Entenda que, para que seu trabalho possa ser publicado, as informações nele contidas não deverão ser ofensivas nem conter material que viole direitos autorais. Seu nome será citado no texto e você aceitará o ônus de ser o único responsável pelo seu trabalho. As contribuições deverão ser enviadas para o endereço eletrônico area8 at dca.ufrn.br (substituindo o "at" pelo caractere "@").
