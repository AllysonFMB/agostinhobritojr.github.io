Algoritmos de traçados de linhas e circunferências

Agostinho Brito

Rasterização

  • A tela gráfica é uma matriz de pontos. Logo, alguns pontos da tela deverão ser selecionados para representar o objeto que se deseja desenhar.
  • O processo utilizado na determinação dos pixels que melhor aproximam um objeto é denominado rasterização.

O analisador diferencial digital

  • Equação diferencial da reta entre dois pontos $(x_1, y_1) e (x_2, y_2)$ \begin{eqnarray} \frac{\,dy}{\,dx} = constante \quad ou \quad \frac{\Delta y}{\Delta x} = \frac{y_2 - y_1}{x_2 - x_1} \end{eqnarray}
  • Aproximação em diferenças finitas:\begin{eqnarray} y_{i+1} & = & y_i + \Delta y\\ y_{i+1} & = & y_i + \frac{y_2 - y_1}{x_2 - x_1} \Delta x, \end{eqnarray}
  • Para implementar um DDA simples, o maior dos valores de \(\Delta x\) ou \(\Delta y\) é escolhido como unidade de rasterização. O algoritmo DDA funciona nos quatro quadrantes.

Algoritmo DDA para o primeiro quadrante

 // Tamanho: Aproxima o tamanho da reta
 if( abs(x2 - x1) > abs(y2 - y1))
      Tamanho = abs(x2 - x1);
 else
      Tamanho = abs(y2 - y1);
 //seleciona o maior dos valores entre Delta_x e Delta_y como unidade de rasterização
 Delta_x = (x2 - x1)/Tamanho;
 Delta_y = (y2 - y1)/Tamanho;
 i = 1;
 while(i < Tamanho){
 //round: valor arredondado de um dado número real. round(-8.6) = -9; round(-8.4) = -8
   setPixel(round(x), round(y));
   x = x + Delta_x;
   y = y + Delta_y;
   i = i + 1;
 }

    • Se a função round for substituída por outras funções de conversão para inteiros, os resultados serão diferentes.

Algoritmo de Bresenham para traçado de linhas

  • Para cada ponto a ser traçado, o algoritmo verifica o erro entre a coordenada y da reta e a linha de rasterização atual.
  • Cálculo da nova coordenada y: \(y_{i+1} = y_i + \Delta y / \Delta_x\)
  • Cálculo do erro: \(e = f(x+1) - f(x) = \Delta y/\Delta x \)
  • A cada iteração: \(e = e + \Delta y / \Delta x \)
  • O erro é inicializado com \( \Delta y / Delta x -1/2 \) para possibilitar verificar somente o seu sinal a cada iteração.
  • Quando o erro ultrapassa \(1/2\), a linha de rasterização é mudada. Faça então \( e = e - 1/2 \).

Algoritmo de Bresenham para traçado de linhas

  • Versão em ponto flutuante 
 m = Delta_y/Delta+x;
 e = m - 1/2;
 for(i=1; i <= Delta_x; i++){
   setPixel(x,y);
   while(new_e >= 0){
     y = y + 1;
     e = e - 1;
   }
   x = x + 1;
   e = e + m;
 }

Algoritmo de Bresenham para traçado de linhas

  • O algoritmo de bresenham pode ser melhorado se a parte fracionária do erro for modificada, passando a utilizar somente aritmética inteira.
  • Revisão da fórmula de cálculo do erro: \[new_e = 2 \Delta x e = 2 \Delta y/ \Delta x * \Delta x = 2 \Delta y \]
  • Fazendo o novo erro igual a \( new_e = 2 \Delta y\) será possível observar as mesmas modificações no seu sinal, sem prejudicar o funcionamento do algoritmo.
  • As modifições são apresentadas no algoritmo inteiro de Bresenham para retas no primeiro octante. 
 new_e = 2*Delta_y - Delta_x;
 for(i=1, i <= Delta_x; i++){
   setPixel(x,y);
   while(new_e >= 0){
     y = y + 1;
     new_e = new_e - 2*Delta_x;
   }
   x = x + 1;
   new_e = new_e + 2*Delta_y
 }

Algoritmo generalizado de Bresenham para os quatro quadrantes

 x = x1;
 y = y1;
 Delta_x = abs(x2 - x1);
 Delta_y = abs(y2 - y1);
 s1=Sinal(x2 - x1);
 s2=Sinal(y2 - y1);
 if(Delta_y > Delta_x){
   Temp = Delta_x;
   Delta_x = Delta_y;
   Delta_y = Temp;
   Troca = 1;
 }
 else{
   Troca = 0;
 }
 new_e = 2*Delta_y - Delta_x;
 for(i=1; i<= Delta_x; i++){
   setPixel(x,y);
   while (new_e >= 0){
     if(Troca == 1){
       //Muda para a proxima linha de rasterização
       x = x + s1;
     }
     else{
       y = y + s2;
     }
     new_e = new_e - 2*Delta_x;
   }

   //Permanece nesta linha de rasterização
   if(Troca == 1){
     y = y + s2;
   }
   else{
     x = x + s1;
   }
   new_e= new_e + 2*Delta_y;
 }

Algoritmo de Bresenham para traçado de circunferências

  • A geração dos pontos é feita apenas para o segundo octante da circunferência e replicados para os demais octantes.
  • O pixel selecionado na figura foi previamente escolhido como o mais adequado. O próximo ponto a ser selecionado para o traçado será o que mais se aproximar da circunferência.

Determinação do ponto seguinte no traçado da circunferência

  • Seja $F(x,y) = x^2 + y^2 - R^2$. $F(x,y)$ vale zero, positivo ou negativo, caso o ponto \( (x,y) \) esteja sobre, fora ou dentro da circunferência.
  • Seja \( d \) a variável de decisão, o valor da função \( F(x,y) \) no ponto central entre os dois pixels E e SE, candidatos à ativação. Se \( (x_p, y_p) \) são as coordenadas do ponto corrente, o ponto central a ser testado terá coordenadas \( (x_p + 1, y_p - 1/2) \).
  • $d_{velho} = F(x_p + 1, y_p - 1/2) = (x_p + 1)^2 + (y_p - 1/2)^2 - R^2$
  • Se $d_{velho} < 0$, \( E \) é escolhido. Logo: \begin{eqnarray*} d_{novo} & = & F(x_p + 2, y_p - 1/2) = (x_p + 2)^2 + (y_p - 1/2)^2 - R^2\\ d_{novo} & = & d_{velho} + (2 x_p + 3) \end{eqnarray*}
  • Se $d_{velho} \ge 0$, \( SE \) é escolhido e o novo valor de d será: \begin{eqnarray*} d_{novo} & = & F(x_p + 2, y_p - 3/2) = (x_p + 2)^2 + (y_p - 3/2)^2 - R^2\\ d_{novo} & = & d_{velho} + (2 x_p -2 y_p + 5) \end{eqnarray*}

Algoritmo de Bresenham para circunferências

  • O primeiro ponto da circunferência é \( (0,R) \).
  • O próximo ponto central cai em \( (x_p, y_p) = (1, R-1/2) \). Logo, \( d=5/4 - R \).
  • O valor de d é incrementado com valores inteiros, sendo apenas testado o seu sinal, de modo que a mudança para \( d = 1 - R \) não afetará o processo de desenho.
  • A função pontosDaCircunferência(): replica os pontos no segundo octante para os octantes restantes.
Algoritmo de Bresenham para circunferências
 x = 0;
 y = raio;
 d = 1 - raio;
 pontosDaCircunferencia(x,y);
 while(y > x){
   if(d < 0){
     d = d + 2*x + 3;
     x = x + 1;
   }
   else{
     d = d + 2*(x-y) + 5;
     x = x + 1;
     y = y - 1;
   }
   pontosDaCircunferencia(x,y);
 }
