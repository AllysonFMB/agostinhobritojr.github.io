2.1. Preparando o ecossistema de desenvolvimento

Como toda linguagem de programação, programas em C++ precisam ser editados de alguma forma. O código-fonte desenvolvido em C++ deve ser armazenado na forma de texto simples. A princípio, qualquer editor de texto pode ser usado para editar C++. Não há problemas em editar seus programas no Word ou BrOffice. Entretanto, a comodidade de usar um ambiente de desenvolvimento integrado para criar aplicações é incontestável. Ambientes integrados permitem, por exemplo, realçar com cores o código escrito, deixando mais agradável a tarefa de programação. Ambientes integrados são capazes de automatizar o processo de organização e compilação do código-fonte, focando a atenção do programador no desenvolvimento de algoritmos.

Sendo assim, o passo zero para o aprendizado da linguagem é preparar seu computador com as ferramentas necessárias para editar e compilar programas escritos em C++.

A propósito, C++ é uma linguagem compilada, de sorte que necessita de uma ferramenta para processar os arquivos que contém código C++ e gerar um programa executável. Outros tipos de linguagens podem ser interpretadas, necessitando de um programa que leia as instruções e as execute, sem gerar um programa executável (Ex: Python), ou meio lá, meio cá, um híbrido de compilação com interpretação, como é a proposta da linguagem Java. Entretanto, deixamos essas discussões para outras esferas no aprendizado de linguagens de programação.

Na qualidade de usuário de sistemas Unix, fico acanhado em sugerir um ambiente de programação descente para usuários Windows. Dos que já mantive contato, o Dev-C++ é uma opção interessante, embora acredite que devam existir outras mais atraentes.

Em ecossistemas Unix, por outro lado, existem várias opções de ambientes de desenvolvimento de código em C++ e todas elas geralmente são integradas ao compilador mais popular na comunidade livre: o GCC/G++. Nesta linha, podem ser sugeridas algumas opções de ambientes que possam interessar aos programadores: eclipse, netbeans, KDevelop e Qt Creator. Não dá para afirmar qual deles é o melhor para escolher, pois geralmente a comodidade do usuário normalmente é o fator mais preponderante. O que muda basicamente de uma proposta para outra (do ponto de vista de um usuário iniciante) é o apelo visual e funcionalidades de integração com o sistema em que o usuário desenvolve. A maior parte das tarefas de edição de texto e código realizo usando o bom e velho emacs e compilo os arquivos gerados em um terminal através de linhas de comando. Quando há necessidade de desenvolver aplicativos com interface gráfica em C++, faço uso da biblioteca QT (http://qt.nokia.com) e utilizo o Qt Creator para escrever código e integrar comodamente toda a criação.

Programa simples desenvolvidos em C++ puro geralmente fazem uso de terminais para exibir os resultados produzidos, inclusive no sistema operacional Windows. Certifique-se apenas de que seu ecossistema consegue compilar o código mostrado no Exemplo 2.1 e mostrar o resultado em algum lugar visível, preferncialmente em um terminal de texto. No sistema operacional Windows, o terminal é mais conhecido como Prompt de Comando.

Este exemplo simples contém poucas mas importantes instruções que deveriam ser entendidas e compiladas no seu ambiente de desenvolvimento. Se não for possível compilar este código ou visualizar o resultado, talvez seja interessante reconsiderar a sua escolha.

Exemplo 2.1. Programa testa-ambiente.cpp para testar sua ferramenta de desenvolvimento.

#include <iostream>

using namespace std;

int main(void){
  char s[40];
  cout << "Digite seu nome: ";
  cin >> s;
  cout << "Seu nome eh" << s << "\n";
  return 0;
}

O programa do Exemplo 2.1 pode ser compilado e executado em um terminal Linux a partir das seguintes linhas de comando:

$ g++ testa-ambiente.cpp -o testa-ambiente
$ ./testa-ambiente

Digite seu nome: Jose
Seu nome é Jose

2.2. Descriçao do programa testa-ambiente.cpp

O programa inicia com uma diretiva de inclusão que insere para processamento o arquivo iostream. A diretiva #include instrui o compilador a usar um pré-processador que lê e processa o arquivo referenciado antes de processar o restante do programa.

#include <iostream>

O arquivo iostream contém elementos importantes para a lide com a entrada e a saída de dados padrão. Entenda-se como entrada e saída padrão os dispositivos que realizam recepção de dados e exibição de resultados para o usuário. Na maioria dos sistemas são relacionados com o teclado e o terminal (monitor), respectivamente, onde um dado programa é executado.

Em seguida, aparece o chamado a mais uma diretiva de compilação: a diretiva using.

using namespace std;

Namespaces permitem ao usuário agrupar elementos de um programa sob a tutela de um nome (o namespace). Por exemplo, suponha que você tivesse a disposição duas funções com o protótipo float media(float,float), mas com comportamentos diferentes. Uma delas calcula a média aritmética, e a outra, a média geométrica. Para evitar reescrever o nome da função, o uso de namespaces poderia servir para discriminar entre uma e outra. Por exemplo, Geometrica::media(3,3) chamaria a função média no escopo do namespace Geometrica, e Aritmetica::media(3,3) chamaria a função média implementada no escopo do namespace Aritmetica. . A diretiva using serve para selecionar que namespace (ou subconjunto dele) deverá ser usado a partir da referida declaração. As funções da biblioteca padrão do C++ são todas declaradas no namespace std, daí a declaração using namespace std; estar presente na maioria dos códigos desenvolvidos.

Por enquanto, basta saber que o uso da chamada da declaração acima evita que os nomes definidos no namespace std possam ser chamados sem usar o prefixo std::. Diversos elementos importantes para o usuário estão definidos dentro desse namespace std, tais como os que permitem realizar entrada e saída de dados. Por isso, é bastante comum tal declaração ser necessária na maioria dos programas em C++.

A função main() vem definida logo em seguida. Assim como em C, todo programa em C++ também inicia pelo chamado dessa função. Sendo assim, apenas uma função main() é permitida por programa.

int main(void){
...
}

As operações de entrada e saída de dados em C++ são bastante diferentes do que se costuma usar na linguagem de programação C. Em C, as funções de entrada e saída são printf() e scanf(), respectivamente. Em C++, duas entidades são usadas para desempenhar esse papel: cin e cout. No exemplo, encontra-se a seguinte declaração:

cout << "Digite seu nome: ";

Quando executado, esse trecho de código imprime na saída o literal "Digite seu nome:". Sem querer adiantar assunto que será apresentado no futuro, inevitavelmente é necessário apresentar cout como o objeto pré-definido do namespace std responsável por receber variáveis, formatá-las e enviá-las para impressão. O símbolo << que aparece logo em seguida indica para onde o literal da direita está sendo enviado. Neste caso, cout. Pelo fato de enviar elementos para cout, o símbolo << é também conhecido como operador de inserção.

Embora o conceito de objeto só seja apresentado mais adiante, não há como apresentar os elementos que permitem ler e escrever sem tocar um mínimo no assunto. Sendo assim, fica registrado que cout é um objeto que serve como ponte para enviar coisas ao terminal de saída. E quanto à função printf()? Também pode ser usada? Sim! Ela é, inclusive, mais rápida para escrever que a solução usando cout. Entretanto, o uso do cout é menos propenso a erros, devendo ser preferido para programação em C++.

Finalmente, as declarações finais realizam a leitura de um literal digitado pelo usuário e reproduz o que foi lido na saída:

cin >> s;
cout << "Seu nome é " << s << endl;

O símbolo endl no final da declaração equivale a uma quebra de linha, surtindo efeito semelhante ao uso do 

"\n"

.

O objeto cin propõe uma funcionalidade semelhante à do cout, mas destinado à entrada de dados. Ele normalmente é conectado à entrada padrão (teclado) e as informações inseridas pelo usuários podem ser enviadas ao programa pelo operador de extração >>, que aparece logo após o cin.

A propósito, os objetos cin e cout podem lidar com mais de um dado na mesma linha de declaração, como ilustra a última linha do exemplo anterior.
